Task: Link existing implemented actions to Menu Bar and Command System.

Goal:
Many actions already exist in UI/store/editor components, but some are not connected to the top Menu Bar yet.
Wire existing actions into the Menu Bar, command registry, shortcuts, and enabled/disabled state.

Do not implement large new features.
Do not redesign the Menu Bar.
Do not rewrite the command system.
Do not duplicate action logic.
Use existing actions/store methods wherever possible.

============================================================
1. Audit Existing Actions
============================================================

First audit the codebase for already implemented actions.

Look for:
- project store actions
- command registry actions
- toolbar handlers
- floating tool handlers
- editor handlers
- mixer handlers
- plugin insert handlers
- MIDI editor handlers
- audio clip handlers

Find actions that exist but are not accessible from:
- Menu Bar
- Command Palette
- keyboard shortcuts

Create a small mapping list before editing.

Expected categories:
- File
- Edit
- Project
- MIDI
- Audio
- Window
- Tools
- Help

Acceptance:
[ ] Existing implemented actions are identified
[ ] No fake menu action is wired to nothing
[ ] Missing actions stay disabled or TODO

============================================================
2. Single Command Path
============================================================

All menu items should call command handlers, not random local handlers.

Preferred flow:

Menu Item
→ commandId
→ command registry
→ existing store/action handler

If command registry already exists:
- add missing commands there
- update menu items to reference command IDs

If no central command registry exists for some actions:
- create a small bridge without large refactor
- keep action logic reusable

Do not duplicate logic inside MenuPanel.

Acceptance:
[ ] Menu actions and shortcuts share command handlers
[ ] Toolbar/menu behavior does not drift
[ ] No duplicate action implementations

12. Keyboard Shortcut Bridge
============================================================

Ensure shortcuts call command IDs, not separate logic.

Shortcuts:
- Ctrl/Cmd+Z Undo
- Ctrl/Cmd+Y or Ctrl/Cmd+Shift+Z Redo
- Ctrl/Cmd+X Cut
- Ctrl/Cmd+C Copy
- Ctrl/Cmd+V Paste
- Ctrl/Cmd+D Duplicate
- Delete / Backspace Delete
- Ctrl/Cmd+A Select All
- Esc Deselect
- V/P/C/G/M/T/A tools
- S split if existing and not conflicting
- Ctrl/Cmd+G Snap if existing

Rules:
- Do not fire app shortcuts while typing in input/textarea/contenteditable.
- Do not double-execute shortcuts from both component and global listener.
- If MIDI editor has local shortcuts, route them through command system where practical.

Acceptance:
[ ] Shortcuts and menu actions share logic
[ ] No duplicate shortcut execution
[ ] Text inputs still work normally
