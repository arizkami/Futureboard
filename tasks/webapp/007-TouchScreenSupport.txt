# 007-TouchScreenSupport.txt

Task: Add Touch Screen and Pen/Stylus Support for Futureboard Studio.

Goal:
Make the DAW usable on touch screens, pen displays, tablets, 2-in-1 laptops, and large touch monitors without breaking mouse/keyboard workflows.

This pass should improve:
- touch input
- pen/stylus input
- pointer event handling
- gesture handling
- touch-friendly hit targets where needed
- fader/knob/slider touch behavior
- timeline/clip editing with touch
- MIDI piano roll touch editing
- mixer touch control
- floating window/dialog touch behavior
- context menu alternatives for touch
- cross-platform Web/Electron behavior

Important:
Do not redesign the whole UI.
Do not make the desktop UI huge.
Do not remove mouse/keyboard shortcuts.
Do not implement mobile layout.
Keep Futureboard compact and professional, but make touch interactions work reliably.

============================================================
0. Core Rule
============================================================

Use Pointer Events as the primary input system.

Supported pointer types:
- mouse
- touch
- pen

Avoid writing separate mouse/touch code paths unless absolutely necessary.

Preferred events:
- pointerdown
- pointermove
- pointerup
- pointercancel
- pointerenter/leave where useful

Use pointer capture for drag interactions:
element.setPointerCapture(event.pointerId)

Acceptance:
[ ] Mouse still works
[ ] Touch works
[ ] Pen/stylus works
[ ] Existing drag interactions do not regress

============================================================

============================================================

Add input capability helpers.

Suggested file:
src/platform/input/inputCapabilities.ts

Detect:
- hasTouch
- hasCoarsePointer
- hasFinePointer
- maxTouchPoints
- prefersReducedMotion
- pointer type from current event

Helpers:
- isTouchLikePointer(event)
- isPenPointer(event)
- isMousePointer(event)
- shouldUseTouchTargets()
- getPointerDisplayName(pointerType)

Implementation:
- navigator.maxTouchPoints
- window.matchMedia("(pointer: coarse)")
- window.matchMedia("(pointer: fine)")
- event.pointerType

Rules:
- Do not rely on user agent strings.
- Hybrid laptops may have both fine and coarse pointer.
- UI should adapt only where needed.

Acceptance:
[ ] Capability helper exists
[ ] Hybrid mouse+touch devices work
[ ] No hardcoded platform assumptions

============================================================
2. Global Touch CSS Foundation
============================================================

Add safe touch CSS rules.

Do not globally disable selection everywhere.

For interactive DAW surfaces:
- touch-action: none where custom gestures are handled
- user-select: none during drag
- -webkit-user-drag: none for draggable/visual assets
- overscroll-behavior: contain where scroll bounce breaks editor interactions

Areas needing touch-action control:
- arrangement timeline
- clip layer
- piano roll
- velocity lane
- mixer faders
- knobs/sliders
- floating window titlebars
- resizable handles

Rules:
- Do not set touch-action: none on text inputs.
- Do not break scrolling inside normal lists/dropdowns.
- Do not prevent browser zoom globally unless necessary for app surfaces.

Acceptance:
[ ] Timeline does not pan page while dragging clips
[ ] Sliders/faders do not scroll page while dragging
[ ] Text inputs still allow text selection/editing

============================================================
3. Hit Target Strategy
============================================================

Keep visual UI compact, but add invisible touch hit zones.

For touch-like pointers:
- increase hit target to 32px minimum where practical
- visual icon can remain 13-16px
- add padding/hit area around small handles
- do not make desktop UI visually huge

Examples:
- clip resize handles: visible 3px, hit target 12-16px
- fader thumb: visible compact, hit target wider/taller
- toolbar icons: visual 14px, button hit box 28-32px
- piano roll notes: minimum touch hit height
- scrollbars/handles: larger touch area

Acceptance:
[ ] Small controls are usable with finger
[ ] Visual density remains DAW-like
[ ] Mouse precision still feels compact
