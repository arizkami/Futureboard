# 006-DawSystem.txt

Task: Build the foundational DAW system layer using WebAudio first.

Goal:
Implement the missing deep DAW system pieces behind the current UI so the app becomes more than visual mock UI.

This pass should create/clean up:
- project state model
- timeline/transport state
- track/clip/note state
- mixer state
- insert/device state
- routing state
- WebAudio engine adapter
- scheduling/playback foundation
- command/action layer
- dirty/save state
- undo/redo foundation where practical
- selectors/helpers so UI components do not duplicate logic

Important:
Use WebAudio for now.
Do not implement WASM yet.
Do not implement native DSP service yet.
Do not implement VST/CLAP hosting yet.
Do not rewrite the entire UI.
Do not redesign visuals.
Do not break existing UI.

This is a system/state/integration pass.

The final architecture should still allow future:
- Web: AudioWorklet + WASM
- Electron: native DSP .exe service + IPC
- Native rewrite: SphereEngine C++

============================================================
0. Core Rule
============================================================

The DAW must have clear separation:

UI components:
- render controls
- call actions/commands
- read selectors
- should not contain deep DAW business logic

Store/system layer:
- owns project state
- owns timeline/mixer/device state
- owns selected state
- owns dirty state
- exposes actions/selectors

Audio engine adapter:
- owns WebAudio runtime behavior
- receives commands from store/actions
- must be replaceable later

Do not let React components directly become the DAW engine.

10. Audio Scheduling Foundation
============================================================

Implement simple audio clip scheduling using WebAudio.

For first pass:
- Only audio clips with loaded buffers need playback.
- MIDI playback can remain placeholder.

Scheduling:
- On play(positionBeat):
  - calculate current time
  - find clips overlapping or after playhead
  - convert clip startBeat to seconds
  - create AudioBufferSourceNode for each clip
  - connect track nodes to master
  - start at correct offset
- On stop:
  - stop all scheduled sources
  - clear scheduled source map
- On seek:
  - stop and reschedule if playing

Handle:
- clip offset
- clip duration
- muted clip
- muted track
- solo state basic
- track volume/pan

Do not implement:
- time stretching
- pitch shifting
- reverse playback unless already easy
- full plugin DSP

Acceptance:
[ ] Imported audio clip can play through WebAudio if buffer is loaded
[ ] Stop stops playback
[ ] Seek updates playback position
[ ] Track fader affects playback
[ ] Mute affects playback
[ ] No crash if buffer missing
