Task: Implement Windowed Floating Panels, Dialog Windows, Electron External Floating Windows, Project Wizard, and Unsaved Changes Dialog.

Goal:
Build a reusable window/dialog system for Futureboard Studio that works across Web and Electron, with optional real external windows in Electron.

This system will support:
- Internal floating panels inside the app
- Internal dialog windows inside the app
- Electron external floating BrowserWindow windows
- Project Wizard
- Unsaved Changes confirmation dialog
- Future plugin floating windows
- Future mixer/editor detached panels

Do not redesign the whole app.
Do not replace the current app shell.
Do not introduce a huge window manager dependency.
Build a small focused system that fits the current DAW UI.

============================================================
1. Window System Concepts
============================================================

Support three window modes:

1. Internal Floating Window
- Works in Web and Electron
- Rendered inside React app
- Draggable
- Resizable
- Focusable
- Z-index managed
- Useful for plugin windows, utility panels, small tools

2. Internal Dialog Window
- Works in Web and Electron
- Modal or non-modal
- Used for Project Wizard, Unsaved Changes, Preferences, Export dialogs
- Has backdrop if modal
- Keyboard handling: Esc, Enter where appropriate

3. Electron External Floating Window
- Electron only
- Creates real BrowserWindow through preload/main IPC
- Used later for detached mixer/plugin/editor windows
- Web fallback should open internal floating window instead

Important:
Web and Electron should share the same high-level window API.
Platform differences must be behind an adapter.

6. Platform Window Adapter
============================================================

Create platform adapter:

src/platform/windows/windowAdapter.ts

Interface:

interface WindowAdapter {
  supportsExternalWindows: boolean;
  openExternalWindow(config: ExternalWindowConfig): Promise<string | null>;
  closeExternalWindow(id: string): Promise<void>;
  focusExternalWindow(id: string): Promise<void>;
}

Web adapter:
- supportsExternalWindows = false
- openExternalWindow returns null
- fallback should open internal floating window

Electron adapter:
- supportsExternalWindows = true
- calls preload IPC:
  window.futureboard.windows.openExternal(...)
  window.futureboard.windows.closeExternal(...)
  window.futureboard.windows.focusExternal(...)

Do not call Electron APIs directly in React.

Acceptance:
[ ] Web fallback works
[ ] Electron path uses preload IPC only
[ ] No direct Node/Electron in renderer

10. Unsaved Changes Dialog
============================================================

Create UnsavedChangesDialog.

Use when:
- New Project while dirty
- Open Project while dirty
- Close Project while dirty
- Quit app while dirty
- Window close in Electron while dirty

Dialog title:
Unsaved Changes

Message:
“You have unsaved changes in this project.”

Actions:
- Save
- Don’t Save
- Cancel

Behavior:
- Save: run save action, then continue pending action
- Don’t Save: continue pending action without saving
- Cancel: abort pending action

Need a pending action mechanism:
type PendingProjectAction =
  | "newProject"
  | "openProject"
  | "closeProject"
  | "quitApp";

or generic callback/ref if safe.

Store/state:
- projectDirty: boolean
- saveStatus: "saved" | "dirty" | "saving" | "error"

Rules:
- Do not lose changes without explicit Don’t Save
- Escape should act like Cancel
- Enter should default to Save or Cancel depending design; safer: Save
- Danger action Don’t Save should be visually clear

Acceptance:
[ ] Dirty project triggers dialog before New Project
[ ] Save continues action
[ ] Don’t Save continues without save
[ ] Cancel aborts
[ ] Clean project skips dialog
[ ] Electron window close can be intercepted if implemented
