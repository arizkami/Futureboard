Task: Add Project Dropdown to Transport Bar with Recent Projects.

Goal:
Add a Zed-like project dropdown when clicking the project name in the Transport Bar / Title Transport Bar.

The dropdown should allow:
- viewing current project
- switching to recent projects
- opening more/local projects
- eventually opening remote projects
- adding opened projects to Recent Projects

Do not redesign the whole transport bar.
Do not change audio engine.
Do not implement full cloud/remote project sync.
Build the UI/state flow first with safe platform fallbacks.

============================================================
1. Transport Bar Project Button
============================================================

Make the project name in the Transport Bar clickable.

Current area:
- Project name
- Save status, e.g. SAVED LOCALLY / UNSAVED

Behavior:
- Clicking project name opens ProjectDropdown
- Dropdown appears below project name
- Button shows active/open state
- Must be app-no-drag so it works inside titlebar
- Text must not wrap
- Long project name should truncate

Visual:
- Zed-like compact dropdown
- dark surface
- subtle border
- soft shadow
- 11-13px text
- current project highlighted
- checkmark on active project

Acceptance:
[ ] Project name is clickable
[ ] Dropdown opens/closes
[ ] Titlebar drag is not broken
[ ] Long project names truncate

6. Switching Recent Projects
============================================================

Clicking a recent project should attempt to open it.

Flow:
1. If current project is dirty, show Unsaved Changes dialog first
2. If user continues, switch/open selected recent project
3. Set current project
4. Move selected project to top of recent list
5. Close dropdown

If full project loading is not implemented:
- switch current project metadata
- show placeholder status
- do not pretend file content loaded if it did not

Acceptance:
[ ] Clicking recent project switches current project or runs safe placeholder

[ ] Recent project moves to top
[ ] Dropdown closes after selection
